# DALOS Cryptography — Genesis Implementation Plan

> Ouronet Core's self-sufficient DALOS Crypto "Gen 1" implementation.
> Replaces the `go.ouronetwork.io/api/generate` dependency with a pure
> TypeScript implementation of the full cryptographic stack. Modular
> design leaves room for future generations (Gen 2, Gen 3, …).

## Vision

1. **End reliance on the Go server.** Today every Ouronet address is generated by `https://go.ouronetwork.io/api/generate` — a remote service running the `StoaChain/DALOS_Crypto` Go code. Private keys generated off-machine, transmitted over HTTPS, trusted by consumers. **We move all of this into the browser/client.**
2. **Port the whole DALOS crypto stack to TypeScript** inside `@stoachain/ouronet-core`. Self-contained, auditable, usable by any JavaScript consumer.
3. **Modular primitive system.** The DALOS cryptography we port is "Genesis" (gen-1). The architecture lets us register a "Gen 2" later without breaking Gen 1 addresses. Think: encryption registry pattern, but for key-generation primitives.
4. **Expose the engine publicly.** Documented API, optionally a standalone npm package, so other projects can adopt DALOS crypto.
5. **Preserve bit-identity with the Go reference.** Existing users' addresses MUST remain derivable from the same inputs — no port can orphan accounts.

## Scope

**In scope:**
- Twisted Edwards curve arithmetic (Dalos Ellipse: `y² + x² = 1 - 26x²y² mod (2^1605 + 2315)`)
- Point operations (Addition, Doubling, Tripling, FortyNiner, PrecomputeMatrix, ScalarMultiplier)
- Key generation from 4 input paths (1600-bit bitstring, base-10 integer, base-49 integer, seed words)
- Seed words → bit-string hashing (Blake3 seven-fold)
- Address string generation (seven-fold Blake3 + 16×16 character matrix)
- AES encryption/decryption for key storage (matching Go's format)
- Schnorr signature signing + verification (present in Go, currently unused on-chain)
- Modular `CryptographicPrimitive` interface
- Registry for multi-generation support
- Integration into the `Codex` class
- UI migration to remove `go.ouronetwork.io` dependency

**Out of scope:**
- Porting DALOS crypto for on-chain signing use (requires Pact runtime changes)
- Formal cryptographic audit by a third party (recommended before any on-chain signing)
- WASM compilation (Phase 11 decides if needed for perf)
- Hardware wallet integration for DALOS keys

## Phase 0 — Audit & Preparation (1 week)

**Goal:** Complete the code audit, lock in test vectors, verify curve parameters mathematically, consolidate Go dependencies.

**Steps:**
1. **Complete Go audit** — I audited Parameters, PointConverter, PointOperations, KeyGeneration, Auxilliary. Still to do:

   - `Schnorr.go` (signing path not used on-chain today)

   - `AES/` (what mode? CBC/CTR/GCM? IV handling? Key derivation path?)
2. **User: re-verify curve parameters in sagemath** — confirm:
   - P = 2^1605 + 2315 is prime
   - Q = 2^1603 + K is prime (where K = 1258387...380413)
   - G = (2, 4795777...) lies on the curve y² + x² ≡ 1 - 26x²y² (mod P)
   - [Q]·G = O (the neutral element, confirming G has order Q)

   - Cofactor R = (P + 1 - T) / Q divides evenly

   - 100× random integer (base 49) → bitstring → pubkey → addresses
   - 50× seed-word combinations (1 word, 2 words, … 12 words, some edge cases with unicode like "héllo wørld") → bitstring → pubkey → addresses
   - 50× Schnorr signatures (message, signer, signature tuple) for sign/verify cross-validation
   - Edge cases: all-zero bitstring, all-ones bitstring, smallest valid seed-word input, largest valid seed-word input
   - Save as CSV + JSON in `StoaChain/DALOS_Crypto/testvectors/`.
4. **Consolidate external Go dependencies.** Today DALOS_Crypto depends on `Crypt0plasm/Cryptographic-Hash-Functions` (separate repo). Fork + merge into DALOS_Crypto so the repo is self-contained:
   - Fork `Crypt0plasm/Cryptographic-Hash-Functions` → `StoaChain/Cryptographic-Hash-Functions` (backup of the original)
   - Move `AES/` and `Blake3/` into DALOS_Crypto as subdirectories
   - Update imports in DALOS_Crypto from `Cryptographic-Hash-Functions/Blake3` → `DALOS_Crypto/Blake3`
   - Verify Go tests still pass
   - Tag as DALOS_Crypto v1.0.0
5. **Write `AUDIT.md` in DALOS_Crypto** — the full audit report lives in that repo for future contributors.

**Deliverables:**

## Phase 1 — TypeScript Port: Math Foundation (2 weeks)

**Goal:** Port the pure-arithmetic layer of DALOS crypto. Every primitive validated against Go test vectors.

**Subpath created:** `@stoachain/ouronet-core/dalos-gen-1/` (gen-1 = Genesis DALOS)

**Steps:**
1. **BigInt modular operations.** TypeScript's `BigInt` handles arbitrary precision; we need modular wrappers matching Go's API:
   ```ts
   class Modular {
     constructor(public readonly p: bigint);
     add(a: bigint, b: bigint): bigint;         // = AddModP
     sub(a: bigint, b: bigint): bigint;         // = SubModP
     mul(a: bigint, b: bigint): bigint;         // = MulModP
     div(a: bigint, b: bigint): bigint;         // = QuoModP (a * b^-1 mod p)
     inv(a: bigint): bigint;                    // modular inverse via extended Euclidean
     exp(a: bigint, e: bigint): bigint;         // modular exponentiation
   }
   ```
   Note: Go's `math/big.ModInverse` uses extended Euclidean. JS BigInt doesn't include this; we implement it ourselves. Straightforward algorithm, ~30 lines.

2. **Coordinate types.** Port as interfaces:
   ```ts
   export interface CoordAffine    { ax: bigint; ay: bigint; }
   export interface CoordExtended  { ex: bigint; ey: bigint; ez: bigint; et: bigint; }
   export interface CoordInverted  { ix: bigint; iy: bigint; iz: bigint; }
   export interface CoordProjective{ px: bigint; py: bigint; pz: bigint; }
   ```

3. **Ellipse definition.** Port `DalosEllipse()` and `E521Ellipse()` as constants:
   ```ts
   export const DALOS_ELLIPSE: Ellipse = {
     name: "TEC_S1600_Pr1605p2315_m26",
     p: 2n ** 1605n + 2315n,
     q: 2n ** 1603n + 1258387060301909...n,  // full value
     t: -5033548...n,
     r: (computed from cofactor formula),
     s: 1600,
     a: 1n,
     d: -26n,
     g: { ax: 2n, ay: 479577721234...n },
   };
   ```

4. **Coord conversions:**
   - `affine2Extended(p: CoordAffine): CoordExtended`
   - `extended2Affine(p: CoordExtended): CoordAffine`

5. **Predicates:**
   - `isInfinityPoint(p)`
   - `isOnCurve(p)` (checks x² + y² ≡ 1 + d·x²·y²)
   - `isInverseOnCurve(p1, p2)`
   - `arePointsEqual(p1, p2)`

6. **Point operations (the bulk):**
   - `addition(P1, P2)` with V1/V2/V3 dispatch
   - `additionV1` (`mmadd-2008-hwcd`): both Z=1 case
   - `additionV2` (`madd-2008-hwcd-2`): one Z=1 case
   - `additionV3` (`add-2008-hwcd`): general case
   - `doubling(P)` with V1/V2 dispatch
   - `doublingV1` (`mdbl-2008-hwcd`): Z=1 case
   - `doublingV2` (`dbl-2008-hwcd`): general case
   - `tripling(P)` (`tpl-2015-c`)
   - `fortyNiner(P)` = 49·P via 3P → 6P → 12P → 24P → 48P → 49P

7. **Testing:** Each operation validated against Go test vectors at multiple test points (not just G). Every formula's branching paths exercised.

**Deliverables:**
- ✅ `@stoachain/ouronet-core/dalos-gen-1/math.ts` with all modular ops
- ✅ `.../coords.ts` with 4 coord types
- ✅ `.../curve.ts` with DALOS_ELLIPSE constant + predicates
- ✅ `.../point-ops.ts` with addition/doubling/tripling/fortyNiner
- ✅ ~80 unit tests matching Go vectors
- ✅ Intermediate milestone: TS can compute `49·G` from `G` and match Go output byte-for-byte

## Phase 2 — TypeScript Port: Scalar Multiplication (1 week)

**Goal:** Port `PrecomputeMatrix` and `ScalarMultiplier` — the base-49 Horner evaluator.

**Steps:**
1. **PrecomputeMatrix:** compute [P, 2P, 3P, …, 49P] via alternating double + add. Mirror Go code's pattern exactly.
2. **ScalarMultiplier(scalar: bigint, P: CoordExtended): CoordExtended:**
   - Convert scalar to base-49 string (using Go's `big.Int.Text(49)` alphabet: `0-9, a-z, A-M`)
   - Base-49 Horner: accumulator = 0, for each digit (MSB first): add PM[digit-row][digit-col], if not last, multiply by 49 via `fortyNiner`
3. **ScalarMultiplierWithGenerator:** shortcut wrapping ScalarMultiplier with generator G as pre-cached point.
4. **Base-49 digit alphabet:** must match Go exactly. Go's `big.Int.Text(49)` uses `0,1,2,…,9,a,b,…,z,A,B,…,M` (49 chars total: 10 digits + 26 lowercase + 13 uppercase A-M). Port this alphabet explicitly.
5. **Constant-time version (stretch goal):** alongside the branching impl, optionally provide a Montgomery-ladder variant for future signing use. Doesn't need shipping in gen-1; leave a TODO.

**Testing:**
- ScalarMultiplier(1, G) == G
- ScalarMultiplier(2, G) == double(G)
- ScalarMultiplier(0, G) == infinity
- ScalarMultiplier(Q, G) == infinity (critical! tests that the base point has order Q)
- 100+ random scalar · G vectors from Go

**Deliverables:**
- ✅ `.../scalar-mult.ts`
- ✅ 100+ tests
- ✅ Critical tests: [Q]·G == infinity, [0]·G == infinity, [1]·G == G

## Phase 3 — TypeScript Port: Hashing + Blake3 Integration (1 week)

**Goal:** Port the hashing layer — Blake3 custom-output-size, seed-word → bitstring, hash → bitstring.

**Steps:**
1. **Blake3 dependency decision:**
   - **Option A (preferred):** use `@noble/hashes/blake3` — mature, audited, supports XOF (variable output). Verify byte-for-byte match against Go's `Blake3.SumCustom` via test vectors.
   - **Option B (fallback):** port the Go Blake3 implementation directly. More work but guaranteed compatibility.
2. **`blake3SumCustom(input: Uint8Array, outputSize: number): Uint8Array`** — wraps Blake3 XOF with specific output length.
3. **`seedWordsToBitString(words: string[]): string`:**
   - Join with single space
   - Convert to UTF-8 bytes
   - Seven-fold Blake3(200 bytes = 1600 bits each round)
   - Convert bytes → hex → bigint → binary → pad with leading zeros to 1600
4. **`convertHashToBitString(hash: Uint8Array, bitLength: number): string`** — generic hex-to-padded-binary.
5. **`affineToPublicKey(p: CoordAffine): string`:**
   - X as decimal string
   - prefixLength = length of X-decimal converted to base-49
   - publicKeyInteger = bigint(X-decimal + Y-decimal)
   - publicKey = publicKeyInteger in base 49
   - return `"${prefixLength}.${publicKey}"`
6. **`publicKeyToAddress(pk: string): string`:**
   - Split on `.`, take the base-49 part, convert to bigint, back to decimal string
   - Seven-fold Blake3(160 bytes = 1280 bits)
   - Map each byte through the 16×16 character matrix (row = byte/16, col = byte%16)
   - Concatenate
7. **`dalosAddressMaker(pk: string, isSmart: boolean): string`:**
   - `Ѻ` prefix for standard, `Σ` for smart
   - Return `${prefix}.${address}`
8. **`CHARACTER_MATRIX`** — literal port of the 16×16 matrix from `KeyGeneration.go`. 256 runes, exactly matching.

**Testing:**
- Seed words → bit string: known inputs from Go test vectors
- Hash → bit string: known hashes from Go
- Public key string encoding: affine → string round-trip
- Address encoding: 100+ vectors (pubkey → address) from Go
- Special characters in addresses: Cyrillic, Greek, Latin extended all round-trip

**Deliverables:**
- ✅ `.../hashing.ts` (Blake3 wrapper)
- ✅ `.../encoding.ts` (public-key ↔ address + character matrix)
- ✅ 100+ tests for address generation
- ✅ Exact byte-level match on all Go address vectors

9. **`generateFromBitString(bits: string): KeyPair`**.
10. **`generateFromInteger(n: bigint, base: 10 | 49): KeyPair`**.

## Phase 7 — Modular Cryptographic Primitive System (1 week)

**Goal:** Wrap the Gen-1 DALOS port behind an abstract interface so future generations plug in cleanly.

**Design — the `CryptographicPrimitive` interface:**

```ts
export interface CryptographicPrimitive {
  /** Stable identifier, embedded in generated addresses. */
  readonly id: string;                  // "dalos-gen-1"
  readonly description: string;         // "DALOS Genesis — 1600-bit Twisted Edwards, Blake3 7-fold, 16×16 char matrix"
  readonly version: number;             // 1 (Gen 1)
  readonly generation: "genesis" | "gen2" | "gen3" | string;

  /** Curve / parameter metadata (for inspection / documentation). */
  readonly metadata: {
    curveName: string;
    primeField: bigint;
    order: bigint;
    cofactor: bigint;
    baseBitLength: number;              // 1600 for Gen 1
  };

  /** Key generation paths. */
  generateRandom(): Promise<KeyPair>;
  generateFromBitString(bits: string): Promise<KeyPair>;
  generateFromInteger(n: bigint, base: 10 | 49): Promise<KeyPair>;
  generateFromSeedWords(words: string[]): Promise<KeyPair>;

  /** Address derivation. */
  publicKeyToAddress(publicKey: string, isSmart: boolean): string;

  /** Given an address string, does it belong to THIS primitive's generation? */
  detectGeneration(address: string): boolean;

  /** Optional: signing (present in gen-1, maybe not in future gens). */
  sign?(keyPair: KeyPair, message: string): Promise<string>;
  verify?(signature: string, publicKey: string, message: string): Promise<boolean>;
}

export interface KeyPair {
  priv: string;                          // base-49 encoded
  publ: string;                          // prefixed base-49 ("N.xxx")
}
```

**Registry pattern (mirrors `EncryptionRegistry`):**

```ts
export class CryptographicRegistry {
  register(primitive: CryptographicPrimitive): void;
  unregister(id: string): void;
  get(id: string): CryptographicPrimitive | undefined;
  detect(address: string): CryptographicPrimitive | undefined;
  all(): readonly CryptographicPrimitive[];
  /** The default primitive for new account creation. */
  default(): CryptographicPrimitive;
}

export function createDefaultRegistry(): CryptographicRegistry;
// Default registry has `DalosGenesis` pre-registered as the only primitive
// and as default.
```

**The Gen-1 implementation wraps the Phase 1-6 port:**

```ts
import * as gen1 from "./dalos-gen-1";

export const DalosGenesis: CryptographicPrimitive = {
  id: "dalos-gen-1",
  description: "DALOS Genesis — 1600-bit TEC, seven-fold Blake3, 16×16 matrix",
  version: 1,
  generation: "genesis",
  metadata: {
    curveName: gen1.DALOS_ELLIPSE.name,
    primeField: gen1.DALOS_ELLIPSE.p,
    order: gen1.DALOS_ELLIPSE.q,
    cofactor: gen1.DALOS_ELLIPSE.r,
    baseBitLength: 1600,
  },
  generateRandom: async () => gen1.keyPairFromBitString(gen1.generateRandomBitsOnCurve()),
  generateFromBitString: async (bits) => gen1.keyPairFromBitString(bits),
  generateFromInteger: async (n, base) => gen1.keyPairFromInteger(n, base),
  generateFromSeedWords: async (words) => gen1.keyPairFromSeedWords(words),
  publicKeyToAddress: (pk, isSmart) => gen1.dalosAddressMaker(pk, isSmart),
  detectGeneration: (addr) => addr.startsWith("Ѻ.") || addr.startsWith("Σ."),  // Gen 1 prefix
  sign: gen1.schnorrSign,
  verify: gen1.schnorrVerify,
};
```

**When Gen 2 exists someday, it registers with its own `id`, `generation`, and `detectGeneration` logic.**

## Phase 10 — Performance Optimization (1-2 weeks, conditional)

**Goal:** If Phase 4 benchmarks show key generation >3s, optimize.

**Decision gates (measure first):**
- If key gen <1s: ship as-is, no optimization needed.
- If 1-3s: add Web Worker wrapping so it doesn't block UI (~2 days work).
- If >3s: consider WASM path.

**WASM options (if needed):**
1. **Option A: Rust port → wasm-pack.** Port critical-path (point ops + scalar mult) to Rust. wasm-pack generates npm-compatible bundle. ~2 weeks. Fastest runtime.
2. **Option B: TinyGo.** Compile the Go code directly to WASM via TinyGo. Potentially least effort (reuse Go code). But TinyGo has compatibility limitations and the output is typically larger than Rust's.
3. **Option C: AssemblyScript.** TypeScript-flavored language that compiles to WASM. Easiest learning curve but modest speedup vs pure JS.

**Deliverables (if needed):**
- ✅ Benchmark results document
- ✅ Web Worker wrapper (always add, low cost)
- ✅ WASM module (if benchmarks demand)

# Risks & mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| Bit-exact reproduction fails somewhere in the port | Users get different addresses, accounts orphaned | Test vectors at every layer; port piece-by-piece with continuous validation; keep Go server as fallback during transition |
| Performance unacceptable | UX degradation on key generation | Phase 10; fall back to keeping Go server for slow clients |
| Curve parameters found unsound in sagemath verification | All existing addresses compromised | Unlikely given prior verification; if found, halt and discuss |
| Blake3 @noble/hashes differs from Go Blake3 | Hash outputs diverge | Audit both against official Blake3 test vectors; port Go Blake3 if needed |
| AES mode in Go isn't Web-Crypto-compatible | Can't decrypt existing key files | Use `@noble/ciphers` for mode flexibility; keep the Go import path for rare legacy files |
| Schnorr audit reveals bugs | Signatures malformed (but not used on-chain today) | Ship gen-1 without signing if broken; fix in gen-1.1 |
| User decides to change curve params mid-port | Wastes weeks of work | Lock in parameters in Phase 0; any change = restart |
